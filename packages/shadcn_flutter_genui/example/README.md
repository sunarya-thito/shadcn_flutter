# GenUI Chat Example

A full chat app that talks to an [OpenRouter](https://openrouter.ai) model and
renders the model's replies as **live `shadcn_flutter` widgets** using
[`genui`](https://pub.dev/packages/genui) and the `shadcn_flutter_genui`
catalog.

- Plain replies show up as chat bubbles (built with `shadcn_flutter`'s
  `ChatBubble`).
- When the model decides to build UI, it emits [A2UI](https://a2ui.org) messages
  that are parsed and rendered inline in the conversation as real shadcn widgets
  (forms, cards, buttons, alerts, sliders, tabs, …).
- User interactions with generated widgets are fed back to the model, so the
  conversation can keep going.

## How it works

```
You type ──▶ Conversation.sendRequest ──▶ A2uiTransportAdapter.onSend
                                                     │
                            _sendAndReceive() POSTs the whole history to
                            OpenRouter's /chat/completions endpoint
                                                     │
              reply text ◀── addChunk() ◀────────────┘
                    │
   A2uiParserTransformer splits it into text + A2UI messages
                    │
        SurfaceController updates surfaces ──▶ Surface widgets rebuild
```

The system prompt (which teaches the model the A2UI protocol and the available
widgets) is generated by `PromptBuilder.custom` from the
`GenCatalog.asCatalog()` catalog. See [`lib/chat_page.dart`](lib/chat_page.dart).

Because `shadcn_flutter` is not a Material app, the GenUI-rendered surfaces are
wrapped in a small Material `Theme`/`Material` host (`_MaterialHost`) so the
catalog's basic widgets that call `Theme.of(context)` keep working.

## Running

```bat
run.bat -d chrome
```

On first launch you land on a **login screen**: paste your OpenRouter API key
(`sk-or-...`) and optionally change the model. The key is saved on-device with
`shared_preferences`, so you stay logged in across restarts. Use **Log out** in
the app bar to clear it and switch keys. Any extra arguments to `run.bat` are
forwarded to `flutter run`, e.g. `run.bat -d windows`.

> Note: `shared_preferences` stores the key in plain text (fine for a demo). For
> a production app, prefer `flutter_secure_storage`.

You can also run it directly:

```bash
flutter run              # then enter the key on the login screen
```

### Choosing a model

Generating valid A2UI JSON is demanding, so prefer a capable free model. The
login screen is pre-filled with `tencent/hy3:free` by default. OpenRouter's free
lineup changes often, so if a model 404s just pick another from
<https://openrouter.ai/models?max_price=0> and type it on the login screen.

To change the default the login screen pre-fills:

```bat
set OPENROUTER_MODEL=meta-llama/llama-3.3-70b-instruct:free
run.bat -d chrome
```
